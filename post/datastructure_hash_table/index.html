[자료구조] 해시 테이블(Hash Table)

    해시 테이블(Hash Table)에 대해 학습한 내용을 정리합니다.


    Hash Table이란?

    • 입력받은 값을 해시 함수의 인자로 넣어서 특정 인덱스로 반환한 후, 해당 인덱스에 값을 저장하는 자료구조를 말한다.
    • EX) input : val, output: 5 -> [0,0,0,0,0,val];

    <간단한 예시>
    const hashTable = [];

function hash(val) {
  return val % 5;
}

function insert(val) {
  const index = hash(val);
  hashTable[index] = val;
}

    Hash의 충돌

    • 해시 함수를 통해 나온 값이 동일한 경우를 말한다.
    • 위의 예시와 같은 경우 해시의 충돌이 일어날 확률이 높다.

    Hash 충돌을 해결하는 방법


    1. 개방 주소법
    • 개방 주소법이란 충돌이 발생할 경우 비어있는 새로운 주소를 찾아 저장하는 방식을 말한다.

    1. 선형 탐사법

      • 충돌 시 해당 데이터 옆공간에 저장하는 방식이다.
      • ex) 해시값이 1이고, 1이라는 공간에 데이터가 있을경우 2에 저장한다.
      • 단점 : 규모가 커질수록 데이터가 밀집해있게 되며, 결국 잦은 충돌이 발생하게 된다.

    1. 제곱 탐사법

      • 충돌 시 데이터의 인덱스의 제곱되는 위치에 저장하는 방식.
      • ex) 해시값이 1이면 1의 제곱, 2라면 2의 제곱 위치에 저장한다.
      • 단점 : 선형 탐사법과 동일하다.

    1. 이중 해시

      • 충돌 시 다른 해시 함수를 한번 더 거치는 방식.

    2. 분리 연결법
    • 값을 저장하는 버킷에 Linked List나 Tree를 사용하는 방식을 말한다.
